Соніфікація карти глибин (depth map sonification) — це технологія, яка перетворює дані про відстані до об’єктів (отримані з датчиків глибини, таких як Kinect, Structure Sensor чи стереокамери) у звукову панораму. Це дозволяє сліпим та людям з вадами зору сприймати оточення через слух: горизонтальне положення об’єкта передається стерео-панорамуванням або binaural-аудіо, а відстань — висотою тону, гучністю, ритмом чи тембром. Такий підхід доповнює тростини чи собаки-поводирі, розширюючи діапазон сприйняття (до 5 м і більше). Дослідження показують, що після кількох хвилин навчання користувачі можуть уникати перешкод і орієнтуватися в приміщенні.
Наукові роботи
Ось ключові наукові публікації (переважно відкриті або з відкритим доступом):

Skulimowski et al. (2019): «Interactive sonification of U-depth images in a navigation aid for the visually impaired». Пропонують систему електронного помічника для навігації. Використовують U-depth — гістограми по колонках карти глибин (аналог U-disparity). Після видалення підлоги та фону карту квантують на 10 діапазонів глибини (до 5 м). Соніфікація: синусоїди з 10 частотами (4000 Гц для найближчого — високий тон, 400 Гц для далекого), гучність пропорційна розміру об’єкта (з нелінійним підсиленням для малих перешкод), стерео-панорамування (лівий канал = ліва частина зображення). Інтерактивність: користувач торкається екрана смартфона, щоб вибрати колонку або отримати вербальну підказку про відстань. Прототип протестовано на 3 сліпих користувачах — успішне проходження коридору з перешкодами.
Brock et al. (2013): «Supporting Blind Navigation using Depth Sensing and Sonification». Використовують Kinect для створення карти глибин. Алгоритм: зменшення роздільної здатності (20×15), marching squares для виділення ізольованих структур перешкод. Найближчі об’єкти соніфікуються: горизонталь → стерео-панорамування, вертикаль → висота тону (вище = вище), відстань → гучність (гучніше = ближче). Звук — модифікований WAV-файл. Формативне тестування на 9 учасниках (8 з пов’язкою на очах) показало швидке навчання (кілька хвилин) і успішну навігацію з мінімальними зіткненнями.
Bologna et al. (2010): «Sonification of Color and Depth in a Mobility Aid for Blind People» (проект See ColOr). Стереокамера генерує карту глибин. Глибина кодується ритмом звуку (тривалість ноти: 160 мс для 0–1 м, до 300 мс для >3 м; для далеких — зменшення гучності за експонентою). Колір соніфікується окремо музичними інструментами (HSL → тембр/висота). Панорама: HRTF (CIPIC database) для binaural-просторового звуку (ліво-право). Тестування на сліпому (з пов’язкою) показало успішне виявлення дверей, шаф і уникнення автомобілів.
Stoll et al. (2015): «Navigating from a Depth Image Converted into Sound». Досліджено пряме перетворення карти глибин у звук для навігаційних завдань. Учасники (з пов’язкою на очах) успішно проходили трасу. Підхід подібний до vOICe, але адаптований під глибину (сканування рядків/стовпців, відстань → параметри звуку).
Сучасна робота (2024–2025): «Listen to Your Map: An Online Representation for Spatial Sonification» (arXiv). Пропонують інкрементальну 360° циліндричну/кругову проекцію карти глибин за допомогою VDB-GPDF + SLAM-подібної локалізації. Сонiфікація: сектори по 10° → «tap»-звуки з BRIR (binaural room impulse responses) для реалістичної панорами, pitch-shift для відстані (нижчий тон = ближче). Перевага — повне 360° покриття (навіть поза полем зору датчика) і стабільність. Тестування показало кращу точність і покриття, ніж сирі дані глибини.

Прототипи
Всі описані роботи реалізовано як робочі прототипи (не комерційні продукти):

Skulimowski: головний датчик Structure Sensor (IR structured light, 640×480, 30 fps) + Android-смартфон + відкриті навушники. Обробка ~25 fps. Інтерфейс — тач-жести на екрані (навіть у кишені).
Brock: Kinect на шиї/голові + портативний ноутбук + навушники. Реальний час, просте обладнання.
See ColOr (Bologna): стереокамера (Bumblebee або STH-MDCS2) + HRTF-конволюція.
Сучасний (arXiv): ROS1 на звичайному ноутбуці (Intel i5), RGB-D сенсор (наприклад RealSense), навушники. Підтримує динамічні об’єкти.

Інші згадки: Microsoft Research (2012) — об’єктна соніфікація з depth-камерами; різні мобільні додатки на базі смартфонів з LiDAR (iPhone) або стерео.
Підходи та алгоритми побудови звукової панорами
Загальна схема (працює в реальному часі на смартфоні/ноутбуці):

Отримання та попередня обробка карти глибин:
Датчик → 2D-масив відстаней (z).
Видалення підлоги (least-squares або RANSAC).
Фільтрація фону (>5 м) і точок вище голови.
Квантування глибини (10–N бінів).

Побудова представлення для соніфікації:
U-depth (Skulimowski): для кожної колонки x — гістограма кількості точок по діапазонах глибини (W×N матриця).
Секторна/360° проекція (сучасні роботи): азимутальний кут + радіальна відстань → кругова/циліндрична растеризація (збереження найближчої точки на кут).
Downsampling + сегментація: marching squares для ізольованих перешкод (Brock).

Мапінг у звук (основні параметри панорами):
Азимут (ліво-право): стерео-панорамування (лівий канал = 1 – x/W; правий = x/W) або HRTF/BRIR для binaural (реалістичне 3D-напрямок).
Глибина (відстань):
Висота тону (pitch): лінійно або обернено (високий = близько, 4000–400 Гц).
Гучність (loudness): пропорційно кількості пікселів або 1/z (з нелінійним boost для малих об’єктів).
Ритм/тривалість: довший звук = далі (90–300 мс).
Pitch-shift або спеціальні звуки («tap» для перешкоди, «woosh» для невідомого).

Розмір/висота: гучність або додатковий pitch (вищий = вище).

Генерація звуку:
Синтез: пакет синусоїд (по 5 мс кожна частота, цикл 50 мс).
Або модифікація WAV + BRIR-фільтр.
Інтерактивність: користувач обирає сектор жестом; авто-режим для близьких об’єктів (<1,5 м).


Приклад алгоритму (спрощений псевдокод з Skulimowski, інтерактивний режим):
textдля кожної колонки x (вибраної користувачем):
  для кожного біну глибини i (0..9):
    a_ie[i] = сума пікселів у сусідніх колонках / висота карти
    a[i] = a_ie[i] * (1 + 1.2 * exp(-8 * a_ie[i]^2))  // підсилення малих
    f[i] = 4000 - i * (3600)/9
  генерувати звук: sum a[i] * sin(2π f[i] t) з часовим вікном
  панорама: vol_L = 1 - x/W; vol_R = x/W
Приклад Brock: усереднення координат перешкоди → panning(x), pitch(y), volume(z).
Переваги сучасних: інкрементальна 360°-карта (VDB-GPDF + pose estimation) дає стабільну панораму навіть поза полем зору датчика.
Висновки та обмеження
Ці підходи доведено ефективними в лабораторних умовах (успішна навігація, низьке навантаження на користувача, навчання за хвилини). Найперспективніші — інтерактивні з 360°-представленням і binaural-аудіо. Обмеження: найкраще працюють indoors (IR-датчики), чутливі до шуму, обмежене поле зору одного датчика (вирішують SLAM/накопичення). Майбутнє — інтеграція з смартфонами (LiDAR), AI-об’єктне розпізнавання та комбінація з голосовими підказками.
